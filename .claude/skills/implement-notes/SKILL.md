# Implement-Notes Skill

將對話中的技術討論內容整理成結構化的 markdown 筆記文件。

## 適用場景

- 程式語言語法和最佳實踐
- 技術概念和運作原理
- 工具使用方法和配置
- 實作邏輯和架構說明
- 問題解決方案

## 執行流程

### 1. 內容分析

- 識別討論的主題和關鍵概念
- 提取程式碼範例和重要說明
- 整理成邏輯清晰的章節

### 2. 檔案命名

- 使用英文 kebab-case 格式：`topic-name.md`
- 範例：`react-hooks-guide.md`、`git-workflow.md`
- 若主題是中文，轉換為英文或拼音
- 避免空格和特殊符號

### 3. 文件結構

1. **標題**: 使用 `# 主題名稱` 格式
2. **摘要**: 簡短說明（1-2 句）
3. **內容章節**: 依討論內容組織
   - 使用清晰的標題層級（##, ###）
   - 程式碼使用 code blocks 並標註語言
   - 重要概念使用粗體或列表
4. **參考資料**（如適用）

### 4. 儲存位置

- 目標路徑：`.claude/skills/implement-notes/references/`
- 執行前先用 Read 檢查目錄是否存在
- 若檔名已存在，詢問使用者：
  - 覆蓋現有檔案
  - 使用新檔名（如：`-v2.md` 後綴）
  - 取消操作

### 5. 完成確認

生成後回報：
- ✓ 檔案路徑：`.claude/skills/implement-notes/references/[filename].md`
- ✓ 內容摘要：簡述記錄的主題

## 品質標準

- ✓ 標題層級清晰（避免跳級）
- ✓ 程式碼區塊標註語言
- ✓ 中英文之間加空格（提升可讀性）
- ✓ 使用 markdown 格式化（粗體、列表、引用等）
- ✓ 避免過度冗長，保持精簡
